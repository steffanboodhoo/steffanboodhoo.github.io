Managing our dependencies and running our app

You may have noticed we haven't linked our index.js in our html page, this was intentional as we're going to have something do that for us. That thing is called webpack

Our dependencies are scattered in our node_modules folder, webpack can be used to easily load these. Also our browser may not be able to directly interpret/run the code we write e.g. JSX or experimental Javascript, webpack can transform our code into a form the browser can understand.

Though I will provide a brief surface level description, I highly advise going through webpack's official documentation. To get started you will need to create a file called webpack.config.js. In this file we export an object that tells webpack (1) where to look to start processing our source code (2) where to output the 'transformed' files (3) what modules to use to interpret and transform the files (4) anything extra

First entry and output, our application starts at index.js, so that shall be our entry point, as for our output it is customary to output to a dist folder, for now create the dist folder in the same directory as the webpack.config.js (but you can output anywhere you like).

Now for the module, this specifies what MODULES are needed to transform which FILES. This is done by (1) using a pattern to match the files we want, (2) the location of which files to not transform (3) the actual modules used to transform these files

We have just listed 3 modules to load/interpret/transform our code, babel-loader style-loader css-loader, we must install these modules, first are the two for css
